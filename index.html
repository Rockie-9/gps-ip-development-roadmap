7 階段建置方法論

Strategic Methodology · 內部品牌變革 IP 開發流程（非消費 IP 模板）

1

策略定錨 Strategic Anchoring

Week 1–2

這個 IP 要替誰、解決什麼、不能是什麼？

對應 Phase 0 · E1–E4 角色檔案 + E5 視覺生成 SOP 前置
交付物 · Deliverables
  • 角色定位陳述 Character Brief（一段話）
  • 使用情境矩陣（3 年所有場景）
  • 競品掃描（含內部既有熊類角色）
⚠ 常見錯誤

直接跳到「畫」而沒有先寫定位陳述，導致設計師交件後發現方向錯了重畫三輪。

★ 決策關卡 Decision Gate

定位陳述若寫不出來，代表前置研究不足 — 不可進入下一階段。本階段為硬性把關，省不得。

2

原型選擇與在地審查 Candidate Selection & Local Review

Week 3–5

承接 19 個候選圖鑑，從「物種研究」進到「角色可行性」

對應 Phase 0 · 候選評估補入既有 E 系列節點之前
交付物 · Deliverables
  • Top 3 候選 SWOT（剪影測試／情緒承載／跨文化轉譯／商標檢索）
  • 在地利害關係人預審紀錄（每候選 1 小時對話）
  • 法務初查報告（TIPO / USPTO / EUIPO / JPO / CNIPA）
⚠ 常見錯誤

跳過在地利害關係人對話。後期被否決的成本是前期諮詢的 50 倍。

★ 決策關卡 Decision Gate

候選須從 19 縮為 1 個主角 + 1 個備案。Director 與 Section Heads 同意主角後進入下一階段。

3

敘事框架 Narrative Framework

Week 6–7

內部品牌變革 IP 必做（消費 IP 通常跳過）

對應 Phase 1 · N1 產品決策框架的內容基礎
交付物 · Deliverables
  • 角色背景故事（500–800 字，所有衍生內容源頭）
  • 聲音與口吻指引（會用什麼字、不用什麼字）
  • 三年敘事整合（Y1/Y2/Y3 角色扮演的角色）
⚠ 常見錯誤

把「設定集」當敘事框架。設定集是 What，敘事框架是 Why — 必須回答角色為何出現。

★ 決策關卡 Decision Gate

背景故事須與 GPS 三年敘事弧線（Y1 changing how security feels / Y2 becomes who we are / Y3 by design）顯式對齊，否則回到階段 1。

4

視覺設計 Visual Design

Week 8–11

大家以為的全部，其實只是 1/7

對應 Phase 0 E5 視覺生成 SOP + Phase 1 治理規範整合
4.1 概念草圖（W8）
  • 3 種風格各 5–8 張縮圖（寫實／插畫／幾何極簡）
  • 剪影測試 24px — 仍能辨認的進下一輪
4.2 主視覺定稿（W9）
  • 三視圖（正、3/4、純側）
  • 表情圖 8 種（中性／歡迎／思考／提醒／慶祝／關心／警覺／引導）
  • 姿態圖 6 種（站／走／招手／指引／坐／伴隨）
4.3 設計系統整合（W10–11）
  • 配色鎖定 GPS 既有 Token（Turquoise #26A7B0 / Copper #CAC3D2 / Black #0A1A22）
  • 比例規範（頭身比、最小尺寸、安全留白）
  • Brand Bible 31 sections 對應更新
⚠ 常見錯誤

一開始就上色。應先做剪影 + 線稿，定形後再進顏色。

5

應用測試 Application Testing

Week 12–13

設計做完不等於可用，要在真實場景驗證

對應 Phase 2a · S7 串聯成效儀表板的前置 baseline
媒材測試
  • 印刷（A4／海報／名片）
  • 螢幕（4K／手機／LINE 貼圖小尺寸）
  • 實體（布章／徽章／立體公仔打樣）
  • 動態（walk cycle／blink／idle）
可用性測試
  • 8–10 位 GPS 同仁（管理師／工程師／主管／行政各層級）
  • 剪影識別、情緒辨識、好感度三項
  • 跨文化檢驗（JP / DE / AZ / SG 各 1 名）
⚠ 常見錯誤

只在桌機看設計檔。實體周邊（布章繡線、印刷網點、3D 立體）會暴露 2D 設計時看不到的問題。

★ 決策關卡 Decision Gate

跨文化測試出現任何單一區域強烈負面反饋，必須回到階段 4 修正。不可「平均分數可接受」就放行。

6

資產生產 Asset Production

Week 14–15

從單一主視覺到完整資產生態系

對應 Phase 2b · I1 多語言機器人 + Phase 3 周邊商品
完整資產包
  • 靜態（PNG / SVG / AI 各尺寸）
  • 動態（Lottie / GIF / MP4）
  • 3D 模型（如有實體周邊）
  • 字體鎖定檔（角色名標準字）
使用指引手冊
  • 30–50 頁 Do/Don't 範例
  • 最小尺寸／淨空／配色／語氣
  • 跨文化版本差異說明
模板庫
  • 簡報、信件、名片、社群貼文、新人訓練各 5–10 套
  • 對應 Brand Bible 既有 31 sections
⚠ 常見錯誤

把「素材包」當交付終點。沒有使用指引，3 個月內各部門就會自由變形產出怪異版本。

7

治理與發布 Governance & Launch

Week 16+ · Ongoing

發布是起點不是終點 — 治理機制決定 IP 能否走完三年

對應 Phase 4 跨部門合作 + Phase 5 活動串聯 + N4 年度審議
發布前 4 週
  • Director 與 Section Heads 預審（最後可否決機會）
  • 分階段發布計劃（內部小範圍 → 全 GPS → 承攬商 → 對外）
維護機制（持續）
  • 誰可新增表情/姿態？誰審核？版權歸屬？
  • 納入既有 GPS Brand Bible 治理流程
  • 年度 IP 審議（對應 N4 既有節點）
★ 決策關卡 Decision Gate

Director 預審若否決 — 在發布前 4 週還來得及修正；發布後再修需付上 10 倍代價。

關鍵原則

Six Principles · 整體流程貫穿的核心紀律

01

內部變革 ≠ 消費 IP

階段 3（敘事框架）與階段 7（治理機制）權重高於外部消費 IP 開發。450 名同仁與 4,000+ 承攬商需要可長期協作的角色，不是促銷吉祥物。

02

剪影優先 · Silhouette First

所有概念必須通過 24px 剪影辨識測試。全球原生物種候選圖鑑工具已用 Pokédex 剪影驗證了 19 個候選的剪影強度。

03

在地審查省不得

後期被否決的成本是前期諮詢的 50 倍。Camas 百合需 Chinookan 部族對話、客家獅需新城社區合作、肥後六花需 JV 四方共識。

04

排除熊類為硬規範

TSMC 已有熊類 IP；含字源熊本（くまモン）、抽象「熊腳印」、薩克森棕熊全部排除。此規範須寫入階段 1 的 Character Brief。

05

三年敘事顯式對齊

Y1「changing how security feels」／Y2「becomes who we are」／Y3「by design」— 角色在每年扮演什麼角色，必須在階段 3 明寫。

06

5 個決策關卡，不可跳過

階段 1, 2, 3, 5, 7 各有硬性決策關卡。任何一關未過不得進入下一階段。壓縮時程會犧牲在地審查與測試 — 後期付出更大代價。
